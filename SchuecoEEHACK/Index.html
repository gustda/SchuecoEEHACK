Challenge: Smart facades for easier maintenance

Statement of the challenge:

How can we support building owners and facility managers in maintaining and cleaning facades via smart building skins?

Facades such as windows and doors allow for a comfortable indoor conditions by controlling the temperature, noise, humidity and light. In order to function properly facades need to be maintained and cleaned. Enabling facades to communicate their conditions supports building owners and facility managers in servicing the walls. Sensors and communication technology allow for smart building skins which can ensure comfortable indoor conditions and high durability of buildings and can deliver information back to the city.

API documentation

This API has been developed to give you an easy way to complete this challenge. It contains simple solutions based on websockets and JSON. You can connect to a server and change all parameters there. The server will push any changes of any parameter to every connected client.This makes it possible for applications to manipulate any Property of the shown example. A complete Solution for a client implementation is shown on the demo client tab. An independently executable solution can simply be copied there.

Set up a connection

To connect to the server, a websocket connection has to be established. The websocket should be addressed with the following URL: ws://schuecobe5hackdays.azurewebsites.net/WebSocketServer.ashx?

A simple and no secure websocket connection is expected

Connect with a property

To change values on the server, a connection to one or more properties is needed. You can connect to a new property by simply sending a "0". If you want to log into a specific property, give its id with the request. Attention: you can also log into a property of another group. The properties are not secured. In case of doubt, please choose a new property. 
                
{
    "type": "connection_request", 
    "request_type": "connect_to_prop", 
    "prop_id": id
}

The server recommends the connection attempt by sending the allocated property id.

Receive data

The server can send data to the clients. For this purpose, it sends a JSON structured like this: 
                
{
    "type" : "property_update",
    "value_name" : "ambient_temperature",
    "value" : 100
}

You can see that this is a very similar structure to the connection request. The JSON type "property_update" is used to push properties from the server to the clients. Each change which is made to a value on the server will be sent to all connected clients. Of course, there are only notifications about the properties of the selected room. There are different data types for each value name, to specify value ranges for the value. There is a listing below, describing the different values that can be received. Write data

If you want to send data to the server, there is a very similar process as receiving data. You have to send a JSON as descriped below. 
                
{
    "type" : "set_value",
    "value_name" : "ambient_temperature",
    "value" : 10.0
}

The value types that can be written, are similar to those that have been described in terms of receiving data. All values that can be received this way can also be changed on the server. So if you want a temperature history, just program it and send the values to the server according to your idea.

        function connectProperty(id) {
            if (id == "")
                id = 0;
            
            myObj = { "type": "connection_request", "request_type": "connect_to_prop", "prop_id": id };

            ws.send(myJSON);

            var myJSON = JSON.stringify(myObj);
